Descripción de la Base de Datos

La base de datos está compuesta por varias tablas interrelacionadas que gestionan empleados, clientes, servicios, citas, horarios y otros elementos relacionados. A continuación, se describen las tablas y sus relaciones:

Roles

ID_Rol (int): Identificador único para cada rol.

nombre (string): Nombre del rol.

Empleados

ID_empleado (int): Identificador único para cada empleado.

nombre (string): Nombre del empleado.

apellidoP (string): Apellido paterno del empleado.

apellidoM (string): Apellido materno del empleado.

telefono (string): Teléfono del empleado.

fotografia (png): Fotografía del empleado.

ID_rol (int): Relación con la tabla de Roles.

Clientes

ID_Cliente (int): Identificador único para cada cliente.

nombre (string): Nombre del cliente.

apellidoP (string): Apellido paterno del cliente.

apellidoM (string): Apellido materno del cliente.

whatsapp (string): Número de WhatsApp del cliente.

fotografia (png): Fotografía del cliente.

Citas

ID_cita (int): Identificador único para cada cita.

ID_cliente (int): Relación con la tabla de Clientes.

ID_registroServicios (int): Relación con la tabla de RegistroServicios.

ID_corteCaja (int): Relación con la tabla de CorteCaja.

ID_horario (int): Relación con la tabla de Horarios.

Servicios

ID_Servicio (int): Identificador único para cada servicio.

nombre (string): Nombre del servicio.

descripcion (string): Descripción del servicio.

costo (double): Costo del servicio.

imagen (png): Imagen del servicio.

RegistroServicios

ID_registroServicio (int): Identificador único para cada registro de servicio.

conceptoAplic (string): Concepto aplicado.

costoAdic (double): Costo adicional.

notas (string): Notas adicionales.

Tintes

ID_tinte (int): Identificador único para cada tinte.

nombre (string): Nombre del tinte.

tono (string): Tono del tinte.

imagen (png): Imagen del tinte.

Tintes_Registros

ID_tinte (int): Relación con la tabla de Tintes.

ID_registroServicio (int): Relación con la tabla de RegistroServicios.

Servicios_Registros

ID_Servicio (int): Relación con la tabla de Servicios.

ID_RegistroServicio (int): Relación con la tabla de RegistroServicios.

CorteCaja

ID_corteCaja (int): Identificador único para cada corte de caja.

ID_cita (int): Relación con la tabla de Citas.

total (double): Total del corte de caja.

fecha (timestamp): Fecha y hora del corte de caja.

Horarios

ID_horario (int): Identificador único para cada horario.

ID_dia (int): Relación con la tabla de Días.

ID_empleados (int): Relación con la tabla de Empleados.

ID_estados (int): Relación con la tabla de Estados.

Días

ID_dia (int): Identificador único para cada día.

fecha (date): Fecha específica.

ID_semana (int): Relación con la tabla de Semanas.

Semanas

ID_semana (int): Identificador único para cada semana.

semana (int): Número de la semana.

Estados

ID_estados (int): Identificador único para cada estado.

estado (string): Descripción del estado.

Relación entre Tablas

La base de datos está diseñada para gestionar eficientemente las citas, servicios, empleados, clientes y las operaciones diarias como los cortes de caja. La estructura relacional facilita la integración y la consulta de datos, asegurando que todas las transacciones y registros estén interconectados de manera coherente.
